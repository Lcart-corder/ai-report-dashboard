# 楽天市場の商品登録フロー分析

参考サイト: https://next-engine.net/ec-blog/rakuten-product-registration/

## 商品登録の3つの方法

1. **RMSから個別登録** - 1つずつ手動で登録
2. **CSVで一括登録** - 複数商品をまとめて登録（有料オプション月額10,000円）
3. **EC一元管理システム** - 複数モールを一元管理

## 商品登録に必要な項目（8ステップ）

### ステップ1: 商品の基本情報
- **商品管理番号**（必須）- 在庫管理用の共通番号
- **商品番号** - 楽天内部の番号
- **商品名**（必須）- 最大127字、推奨30字以内
  - 前半に重要なキーワードを配置
  - 例：【ブランド名_商品名称_対象性別_シーズン_仕様_色_サイズ】
  - 具体例：【タイトスカート 裏地付き 全5色 M/L】

### ステップ2: 販売・価格情報
- **販売価格**（必須）- 実際の販売価格
- **表示価格** - メーカー希望小売価格や通常価格（二重価格ガイドライン遵守）
- **商品別ポイント変倍** - 商品ごとのポイント倍率設定
- **商品オプション** - サイズ・カラーなどの選択肢

### ステップ3: 在庫・配送情報
- **在庫数**（必須）
- **配送情報**
- **最短お届け可能日表示** - 楽天最強翌日配送ラベル取得に重要

### ステップ4: 製品情報
- **ジャンルID**（必須）
- **カタログID**（必須）
- **JANコード** - ID検索・JANコード検索で選択

### ステップ5: ページデザイン
- **商品説明文** - ユーザーに分かりやすく魅力的な説明
- **商品写真** - コンバージョンに大きく影響

## 重要なポイント

1. **商品名の最適化**
   - 30字以内推奨
   - 前半に重要キーワード配置
   - PC・スマホ両方で見やすく

2. **二重価格表示**
   - 根拠のある表示価格のみ使用
   - ガイドライン遵守

3. **配送品質**
   - 最短お届け可能日の設定が重要
   - 楽天最強翌日配送ラベル取得に影響

4. **商品写真**
   - コンバージョンに直結
   - 複数枚対応

5. **商品オプション（バリエーション）**
   - サイズ・カラーなどの選択肢
   - ユーザーの購入体験向上

## Lカート（エルメ）への適用

現在のLカートシステムに以下を実装すべき：

1. **商品登録の8ステップウィザード**
   - 基本情報 → 価格設定 → 在庫・配送 → 製品情報 → ページデザイン → オプション → プレビュー → 公開

2. **商品名の自動最適化**
   - テンプレート提供
   - 文字数カウンター
   - キーワード配置のガイド

3. **商品画像の複数枚対応**
   - メイン画像 + サブ画像（最大10枚程度）
   - ドラッグ&ドロップアップロード
   - 画像編集機能

4. **バリエーション設定**
   - サイズ・カラー・その他オプション
   - 在庫・価格の個別設定
   - SKU管理

5. **CSV一括登録**
   - テンプレートダウンロード
   - バリデーション機能
   - エラー表示

6. **プレビュー機能**
   - 実際の商品ページのプレビュー
   - デスクトップ・モバイル表示切り替え

7. **SEO設定**
   - メタタイトル
   - メタディスクリプション
   - URLスラッグ

8. **在庫管理**
   - 在庫数の設定
   - 在庫切れ時の表示設定
   - 在庫アラート
